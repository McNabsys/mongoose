**Status:** 2026-04-26. Direction C v2 working (7.15% on single cache), Direction A
killed (warmstart-fade collapse). Beating production is the bar.

## What we know about physics-model errors

Combined evidence from Phase 0a residual decomposition + the 34 Nabsys
Confluence docs:

| # | Error | Magnitude | Production solution | V3/V4 has it? |
|---|---|---|---|---|
| 1 | **Head dive** (first ~40 kb) | ~10× residual lift | Method 1: per-molecule severity (probe-width fit + count) × 16 length-group splines | No |
| 2 | **Tail effects** (last ~10 kb) | Bandwidth-limited; interval-error 1.08-1.24× near tail | Method 1 separate tail-pass with fixed severity = 1.0 | No |
| 3 | **Velocity decile spread** (slow molecules ≠ fast) | 14× spread D1→D10 in T2D residual | Per-channel T2D fit + per-detector-condition TVC dual-exponential bp shift | No (V3 has probe-aware velocity head, didn't help) |
| 4 | **`attr_in_structure` / `attr_folded_start`** | 10× residual lift | RTS branch + MF/PF filters with analytical 831 bp expected width | **No — these molecules are FILTERED OUT** at preprocess.py:142 |
| 5 | **Per-detector calibration variation** | Frame A vs B vs C × V/psi → different TVC parameters | Per-detector × per-V × per-psi calibrated parameter table | No (V4-C v1 model-vs-prod gap of 1.8-4.4% across runs is consistent with this) |
| 6 | Stretch distribution | r ≈ 0 against T2D residual (refuted as primary axis) | Per-molecule stretch scalar via remap | Not load-bearing |
| 7 | Calibrant artifact | Lambda vs human → Mult/Alpha shift | Lambda hardcoded as production calibrant | We use the same data |

**Refuted from earlier hypotheses:**
- Blue holdout's lower velocity CV is NOT a genuine signal — likely a
  Lambda-calibrant artifact (per the human-vs-Lambda Confluence doc).
- `molecule_stretch_factor` r ≈ 0 — single-scalar stretch is not informative.

**The four load-bearing axes for V5:** head-dive (#1), tail (#2), velocity
decile spread (#3), and structured-region exclusion (#4). All four are
things production's HDM stack handles explicitly via parameterized
corrections that V3/V4 don't have.

---

## Constraint

**Detector ID, voltage, pressure cannot be model inputs at inference time.** This rules out the obvious "just add detector_model_id as a categorical feature" solution. We need detector-INVARIANT features that capture per-detector variation through molecule-intrinsic measurements (rise times, fall times, level-1 amplitude, probe-width distribution moments, etc.).

## What V4 taught us

| | Direction A (noise-model loss) | Direction C v1 (production_residual MLP) | Direction C v2 (ref_anchored Huber) |
|---|---|---|---|
| Status | Killed at epoch 9 | Plateaued | Smoke confirmed |
| Result | 28.4% holdout median (vs T2D 16.2%) | 8.07% interval rel-err | 7.15% on single cache |
| Diagnosis | Warmstart-fade transition collapses probe head | Capped by production (target = production output) | Improving over v1 but architectural ceiling at ~7% |
| Key learning | Noise-model NLLs are unstable when probe head supervision drops | Per-probe MLP w/ engineered features beats T2D | Ref-anchored target gives access to "beat production" signal |

**The central V4 finding:** the per-probe MLP on (probes.bin features + molecule aggregates) extracts most of production's correction signal but **cannot beat production** because:
1. The features are post-wfmproc — same information production already used.
2. The architecture is per-probe — no inter-probe context to learn TVC's interval-size dependence directly.
3. The current data pipeline filters out the high-residual population production designed RTS for.

**To beat production, V5 needs to do at least ONE of:**
- Use information production discards (raw waveform — V4-A's promise, before training broke)
- Use richer cross-probe context (sequence model, length-group splines)
- Train on the population production handles via filters but V3/V4 currently exclude (RTS region)
- Train against the GENOME directly, not production's output (Direction C v2's framing)

## V5 plan — three phases, in order of risk

### Phase 1: V5-Lite (1-2 weeks, $0-30 cloud cost)

**Goal:** push Direction C v2 (~7%) to *beat* production (5.77%) by adding the cheapest physics-aware extensions.

Three concrete changes, each independently testable:

**1.1 Open the data filter (Error #4)**

`preprocess.py:142` currently drops molecules with `structured / folded_start / folded_end / do_not_use`. Phase 0a says these molecules carry the 10× residual lift. Production handles them via the RTS branch. **V5 includes them.**

Implementation:
- Modify preprocess.py to keep these molecules with their attribute flags.
- Add an `is_structured_molecule` boolean to the residual table (already in attr bit-field).
- Re-cache (~30 min per run × 28 runs = 14 hours background, or 1.5 hr on cloud).
- **Critical:** they need their own evaluation track too, since current bp-interval rel-err only includes accepted+matched probes.

Expected value: directly addresses Phase 0a's biggest single residual axis. If the model gracefully handles this population, 1-2 percentage points of bp-interval improvement.

**1.2 Detector-invariant fingerprint features (Error #5)**

Add to per-molecule feature broadcast (constant across all probes of a molecule, no detector ID):
- `rise_t10`, `rise_t50`, `rise_t90` (already parsed by `Molecule` dataclass)
- `fall_t90`, `fall_t50`, `fall_t10`
- Ratios: `(rise_t90 - rise_t10) / transloc_time_ms` — sharpness metric
- `mean_lvl1_from_tdb` (already in conditioning)
- Probe-width-distribution moments: `std`, `skew`, `kurtosis` of the molecule's probe widths

These vary systematically by detector but are measured per-molecule, so the model can self-discover the conditioning without knowing the detector ID.

Implementation: extend `reads_maps_table.py` to load `probes.bin` for fingerprint fields, plus `extract_features` in `residual_dataset.py` to add ~8 new numeric features. ~2 hours. FEATURE_DIM goes 22 → 30.

Expected value: closes most of V4-C v1's ~2% per-run model-vs-prod gap if the gap is per-detector-driven (which the variance pattern suggests).

**1.3 Length-group conditioning (Error #1, head-dive)**

Already have `length_group_bin` (16 bins, 75-235 kb). Currently passed as a raw integer. Replace with:
- 16-dim learned embedding (model picks how to use the bin)
- Plus interaction term: `frac_position_in_molecule × length_group_embedding`

This explicitly mimics production Method 1's structure: each length-group has its own correction curve along the molecule. The model can now learn 16 distinct curves.

Implementation: ~1 hr. Add an embedding module to `ResidualMLP`'s stem.

Expected value: addresses head-dive (#1) directly. Should improve decile-9 (the +5 kb residual tail).

**Phase 1 deliverable:** V5-Lite — V4-C v2 architecture + (1.1, 1.2, 1.3). Expected:
- Held-out bp-interval rel-err: 5.5-6.5% (target: matches or beats production's 5.77%)
- Per-decile improvements: better tail accuracy (head-dive), better attr_in_structure handling
- Trained overnight on local A4500, no cloud cost beyond current spend

### Phase 2: V5-Sequence (3-4 weeks, $50-150 cloud cost)

**Goal:** structural lift via inter-probe context. Run only if Phase 1 doesn't beat production.

**2.1 Per-molecule sequence model**

Replace per-probe MLP with a small transformer or LSTM operating on the per-molecule probe sequence:
- Input: variable-length sequence of (per-probe features), padded to max-K
- Each "token" is a probe's feature vector
- Output: per-probe predicted shift

This lets the model directly learn TVC's interval-size dependence (the model can compare a probe to its neighbors), and head-dive's positional dependence (early-in-sequence probes get different treatment than late).

Implementation effort: ~1 week. Architecture: 4-layer transformer encoder, embed_dim 128, ~2-5M params.

Expected value: structurally addresses #1 and #3. If V5-Lite plateaus near production but doesn't beat it, this is where inter-probe context starts mattering.

**2.2 Train against the genome (Direction C v2 framing)**

Use `ref_anchored_residual_bp` target with Huber loss (validated in v2 smoke). The sequence model can use cross-probe context to estimate per-molecule stretch + per-probe correction simultaneously.

**Phase 2 deliverable:** V5-Sequence. Expected: 5.0-5.7% bp-interval rel-err, beating production by 0-1 pt. If it doesn't beat production, the sequence-model framework still produces a strong V6 starting point.

---

### Phase 3: V5-Waveform (4-6 weeks, $200-400 cloud cost)

**Goal:** access information production discards. Last resort if Phases 1-2 don't beat production.

**3.1 Fix Direction A's warmstart-fade collapse**

V4-A's failure mode was specific: probe head (centernet focal) trained during warmstart, then collapsed when blend faded to 0 because the noise-model bp/vel/count NLLs pull in different directions than peakiness regularizer. Fix candidates:
- **Min-blend floor of 0.1-0.3** — keep centernet supervision throughout, never fully fade
- **Sequential warmstart** — train probe head fully (frozen), then unfreeze with bp/vel/count
- **Re-derive the peakiness regularizer** to match what centernet would produce

**3.2 Compose with V5-Lite/V5-Sequence**

The V4-A waveform U-Net's output (predicted cumulative bp + probe heatmap) becomes an input to the V5 per-probe / per-sequence head. The waveform U-Net captures signal production's wfmproc discards (raw peak shapes, sub-probe transitions, baseline drift). The V5 head refines that prediction using engineered features.

**3.3 Loss against genome**

Same Direction C v2 framing — train against reference, not production. Composes cleanly.

**Phase 3 deliverable:** V5-Waveform. Hardest, biggest potential. Could beat production by 1-3 pts if waveform-side signal is real.

## Decision points

Before each phase ends, the question is: **does it beat production on held-out bp-interval rel-err?**

- If V5-Lite beats production → done with this question. Move to V6 / scaling up training.
- If V5-Lite doesn't beat but closes to within 0.5 pt → likely diminishing returns; document and pivot to V6 (different architecture entirely).
- If V5-Lite doesn't move at all → revisit the "per-probe MLP architecture is structurally limited" hypothesis. Skip to Phase 2 or Phase 3.

---

## What this plan deliberately does NOT include

- Per-channel detector ID. Constraint says no.
- Adding voltage / pressure as inputs. Same constraint.
- Re-running V4-A as designed. Warmstart-fade collapse is a real bug, not a hyperparameter problem.
- Full re-architecture from scratch. V4-C is the working baseline; we extend it.
- Anything that requires re-running production calibration (the M1 spline fits, the TVC dual-exp coefficients). We mimic, not re-derive.
