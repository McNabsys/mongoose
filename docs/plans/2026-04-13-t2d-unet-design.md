# T2D U-Net Architecture Design

**Project Mongoose** -- Deep learning replacement for the legacy physics-based Time-to-Distance model.

## Problem Statement

The legacy T2D model (`L = C * (t + offset)^alpha`) uses a single-parameter viscous drag formula per channel to convert temporal nanochannel signals to spatial (base pair) coordinates. Empirical analysis of 31,468 clean E. coli molecules shows trailing-end velocity is 3.5x faster than mid-molecule velocity, with a head-dive acceleration burst at the leading edge. The nonlinear fluid dynamics (TVC, head dive, channel drift, concentration polarization) exceed what static parametric equations can capture.

## Reference Data

- **Organism:** E. coli K-12 MG1655 (NC_000913.3), 4,641,652 bp, circular
- **Enzyme:** BssSI only, ~810 probe sites
- **Tag invariant:** 511 bp physical width per tag
- **Training data:** 30 E. coli runs, 3 instruments (202, 429, 433), 5 dies (H09, B23, G16, C12, D08), wafer L58270w05. ~3.2M total events, ~1.8M clean.

## Ground Truth Construction

### Shift-Invariant Inter-Probe Deltas

Training targets are **inter-probe distances in base pairs**, derived entirely from the E. coli reference genome. This avoids leaking legacy T2D bias through estimated molecule tail lengths.

1. From the `.assigns` file: get the reference probe indices mapped to each molecule, plus direction (+/-)
2. From the `_referenceMap.txt`: look up absolute bp positions of those reference probes
3. Sort by temporal order in the waveform
4. `GT_deltas = abs(diff(probe_bp))` -- always positive, handles forward/reverse natively
5. Tag-width velocities (`511 / duration`) stored separately as auxiliary velocity targets

### Why Not Absolute Positions

The physical molecule ends do not land on restriction sites ("blind overhangs"). The `.assigns` file's estimated Ref_Start/Ref_End are generated by the legacy T2D model. Using `torch.diff` on the predicted cumulative curve makes the loss shift-invariant to unknown tail lengths and localizes gradients to each interval independently.

## Architecture

### Overview

A 1D Physics-Informed Neural Network (PINN) U-Net. The network predicts per-sample **instantaneous velocity** (bp/sample), which is deterministically integrated via `torch.cumsum` to produce a monotonically increasing cumulative base-pair curve.

### Encoder (5 levels)

| Level | Channels | Downsampling | Spatial Dim (4000-sample mol) |
|-------|----------|--------------|-------------------------------|
| 0     | 32       | 2x MaxPool   | 4,000 -> 2,000                |
| 1     | 64       | 2x MaxPool   | 2,000 -> 1,000                |
| 2     | 128      | 2x MaxPool   | 1,000 -> 500                  |
| 3     | 256      | 2x MaxPool   | 500 -> 250                    |
| 4     | 512      | 2x MaxPool   | 250 -> 125                    |

Each level: 2x ResBlock(kernel_size=7, BatchNorm, ReLU).

FiLM conditioning injected at Level 0 and Bottleneck (see Conditioning section).

### Bottleneck (512 channels)

Operates on 125-625 tokens depending on molecule length.

1. Dilated convolution cascade: 4x ResBlock with dilation rates [1, 2, 4, 8], kernel_size=7. Pushes theoretical RF past 4,500 samples.
2. 1D Multi-Head Self-Attention (4-8 heads). Gives every token global context across the entire molecule. Computationally trivial at 125-625 tokens.

This handles the worst-case scenario: a false-negative-expanded 20 kbp desert at 200 kbp/s velocity = 4,000 samples.

### Decoder (5 levels, symmetric)

Each level: Upsample 2x, concatenate skip connection, 2x ResBlock(kernel_size=7). Channel counts mirror encoder (512 -> 256 -> 128 -> 64 -> 32).

### Bifurcation at Full Resolution (1x)

Both branches operate at full temporal resolution after the shared decoder.

**Probe Branch:**
- 2x ResBlock(kernel_size=7) -- sharp, high-frequency boundary detection
- Conv1d(32, 1, 1) -> Sigmoid
- Output: probe center heatmap, values in (0, 1)

**Velocity Branch:**
- 2x ResBlock(kernel_size=31) -- wide kernels for learned low-pass smoothing
- Conv1d(32, 1, 1) -> Softplus (ensures strictly positive velocity)
- Multiply by `molecule_mask` (zeros in padded regions)
- `torch.cumsum(masked_velocity, dim=-1)` -> cumulative bp curve

Wide kernels (k=31) smooth baseline noise while preserving the localized velocity dips caused by tag hydrodynamic drag. The cumsum operator itself acts as a natural low-pass filter.

### Why Not Bifurcate Earlier

At 8x downsampling, a trailing-end 511bp tag (13.6 samples) compresses to 1.7 tokens. The velocity branch's auxiliary L2 loss targets these exact sample indices. Upsampling from 8x smears the tag-induced velocity dip across 32 samples, causing gradient conflicts with the localized loss. Both branches must reach full resolution.

## Conditioning (FiLM)

A 6-value continuous physics vector, all measurable per-molecule without T2D:

1. **Absolute pre-event baseline** (raw, unnormalized) -- buffer conductivity, pore diameter proxy
2. **log(molecule duration in samples)** -- macro-scale drag prior from polymer physics
3. **log(inter-event interval + 1)** -- concentration polarization (rested vs exhausted pore)
4. **Time-in-run** (fraction 0.0 to 1.0) -- Joule heating, viscosity drift
5. **Applied bias voltage** -- velocity proportional to electric field
6. **Applied pressure** -- pneumatic back-pressure

```
conditioning [6] -> Linear(6, 64) -> ReLU -> branch:
    -> Linear(64, 2*32)  -> gamma_0, beta_0  (Encoder Level 0)
    -> Linear(64, 2*512) -> gamma_b, beta_b  (Bottleneck)

FiLM: output = gamma * features + beta
```

No categorical channel IDs. Level-1 normalization of the input waveform neutralizes the dominant cross-channel geometric variance. The continuous FiLM vector captures residual channel-state variation and generalizes to unseen detectors.

## Loss Function

```
L_total = L_probe + lambda_1 * L_bp + lambda_2 * L_velocity
```

### L_probe (Focal Loss)

- Target: soft Gaussian heatmap centered on each probe
- Dynamic sigma: `sigma_i = max(1.5, duration_samples_i / 6)` -- bounds 99% mass within measured tag width
- Focal loss with gamma=2, alpha=0.25 to handle extreme class imbalance (~9% positive)

### L_bp (Sparse Huber on Inter-Probe Deltas)

- `pred_bp = cumsum(pred_velocity * molecule_mask)`
- `pred_probes = pred_bp[probe_sample_indices]`
- `pred_deltas = diff(pred_probes)`
- `GT_deltas = abs(diff(reference_probe_bp_positions))`
- Huber loss in native base pairs with delta=500bp
- Normalize by dividing the scalar loss output by mean GT interval size

### L_velocity (Sparse L2 at Tag Positions)

- At each detected probe sample index: `target_v = 511 / duration_ms * sample_period`
- L2 loss on `pred_velocity[probe_indices]` vs velocity targets
- Also evaluated at molecule edge regions using rise/fall dynamics from TDB

### Loss Warmup

End-to-end multi-task from epoch 1. Linear warmup of lambda_1, lambda_2 from 0.0 to 1.0 over first 3-5 epochs. No sequential curriculum (causes encoder feature starvation).

## Training

- **Optimizer:** AdamW, lr=1e-3, weight_decay=1e-4, cosine annealing to 1e-6
- **Batching:** Dynamic bucketing by molecule length. Pad within buckets. ~8-16 molecules per batch.
- **Epochs:** 50-100
- **Training set:** Clean, remapped molecules with 8+ accepted probes, translocation time > 30ms. Exclude initial ~2min calibration TDB file per run. ~600K-800K molecules.

### Data Augmentation

- Random Gaussian noise injection (scaled to baseline RMS)
- Random amplitude scaling (+/- 5% of level-1)
- **Uniform time-stretching:** k in [0.90, 1.10]. Interpolate waveform by k. Divide L_velocity targets by k. L_bp targets unchanged (time * velocity = distance). Simulates voltage/viscosity drift.
- **No time-flipping** -- entry/exit geometry is asymmetric (head dive vs trailing whip)

### Train/Val/Test Split (by die)

- **Train:** H09 (inst 433) + B23 (inst 433) + G16 (inst 202) -- ~24 runs
- **Val:** D08 (inst 429) -- 2 runs
- **Test:** C12 (inst 429) -- 1 run

## Inference Pipeline

### Step 1: Forward Pass

Input: level-1-normalized waveform [T] + 6-value conditioning vector.
Outputs: probe_heatmap [T], pred_velocity [T], pred_cumulative_bp [T].

### Step 2: Velocity-Adaptive NMS

- Threshold heatmap at minimum confidence (e.g., 0.3)
- NMS with adaptive window: `min_sep = max(8, 511 / pred_velocity_local * 0.5)` samples
- Translates static 511bp spatial constraint into dynamic temporal window

### Step 3: Sub-Sample Probe Localization

- Fit parabola to heatmap[t-1], heatmap[t], heatmap[t+1]
- `t_frac = t + (p[t-1] - p[t+1]) / (2 * (p[t-1] - 2*p[t] + p[t+1]))`
- `probe_bp = lerp(pred_cumulative_bp[t], pred_cumulative_bp[t+1], frac)`
- Eliminates +/-18bp quantization error at trailing-end velocities

### Step 4: Probe Width as Cluster Classifier

- `duration_bp = pred_cumulative_bp[t + half_width] - pred_cumulative_bp[t - half_width]`
- ~511bp = clean single tag; ~1020bp = unresolved double-tag cluster

### Step 5: Output Format

- MVP: write `_probes.txt` text format (decoupled from C++ binary serialization)
- Production: load original `probes.bin`, overwrite spatial columns with DL predictions

### Step 6: Quality Gating

- Gate on absolute physical limits: reject if macro-velocity exceeds theoretical max electrophoretic speed
- Do NOT compare against legacy T2D (would reject the model's best successes)

## Evaluation

### Phase 1: ML Metric

On held-out die D08: compute inter-probe interval MAE vs E. coli reference genome for both DL model and legacy T2D. DL model should show lower MAE and tighter variance.

### Phase 2: Business Metric

Run full downstream remapping pipeline with DL model output. Feed molecules that the legacy T2D failed to map through the DL model. Measure "rescue rate" -- increase in usable mapping yield represents effective throughput improvement via software alone.
