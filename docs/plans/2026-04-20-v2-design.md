# V2 Design: Backbone + Loss Composition

**Status:** Complete design, cleared for implementation by Deep Think's full-document review (2026-04-20). All five pre-implementation defects identified in that review (L_heatmap_sharpness inversion, L_velocity/FWHM contradiction, Sinkhorn NaN trap, PixelShuffle1d non-existence, tensor shape/dim mismatches) are resolved in this revision. See Section 10 review history for the full trajectory.

**Relationship to prior designs:**
- Supersedes `2026-04-15-t2d-unet-design-v2.md` for architectural scope only. That document defined the V1 rearchitecture currently in training on branch `claude/peaceful-rubin-bfb7a9`.
- This document defines **V2 proper**: the generation that removes V1's residual wfmproc dependencies, adds a segmentation head, and replaces the CNN U-Net backbone with an 8x-Resolution Transformer U-Net.
- V1 rearchitecture (the Micro Phase 6 work) remains the fallback if V2 runs into training trouble.

## 1. Executive Summary

V2 is the generation of the Mongoose T2D model that:
1. Eliminates wfmproc peak positions as *positional* scaffolding for training (they remain as fold/structure *segmentation* supervision, which is a different task).
2. Learns to spatially segment each molecule into clean vs. defect regions, and emits probe detections only from clean regions.
3. Replaces V1's CNN U-Net backbone with an 8x-Resolution Transformer U-Net that provides fine-resolution bidirectional global reasoning at the Nyquist-adequate token scale.

### V2 Goals (priority order)

1. **Wfmproc-independent peak localization.** The model must not depend on wfmproc probe centers as positional supervision (no warmstart Gaussians on wfmproc positions, no teacher-forced read-outs at wfmproc peak times).
2. **Probe-vs-non-probe discrimination.** The model must discriminate real probes from free tags and other signal structures using reference alignment as the only positional supervision. Rejected probes from wfmproc provide negative anchors during a short warmstart phase.
3. **Structured/folded molecule coverage.** The model must match or beat V1 on reference-bp accuracy on held-out structured/folded molecules by explicitly identifying defect regions and masking peak-detection supervision accordingly.

### What carries forward from V1 rearchitecture
- `cumulative_bp = cumsum(softplus(raw_velocity))` output structure (monotonic by construction).
- Reference-alignment-based supervision from `assigns.bin` + `reference_map.bin`.
- FiLM conditioning on continuous physical observables (expanded from 6 to 12 values).
- Soft-DTW-style alignment loss structure (replaced internally with Sinkhorn-OT, see section 6).
- Self-supervised tag-width velocity **constraint** (`511 bp / FWHM_ms`) — preserved as a Section 8 diagnostic sanity check, not as an explicit training loss (see section 6.4).

### What V2 changes
- Backbone: CNN U-Net (V1) → 8x-Resolution Transformer U-Net with RoPE + AdaLN + FlashAttention.
- Preprocessing filter: relaxed to include structured/folded molecules with spatial defect extents carried as segmentation labels.
- New output head: per-sample 6-class segmentation.
- L_bp: hard-NMS + soft-DTW (V1) → bidirectional Sinkhorn-OT with soft-min over forward/reverse orientations.
- L_probe_anchor: focal-on-Gaussians (V1) → wide-boxcar positive + narrow negative anchors, faded to zero over ~5 epochs.
- L_count: static target (V1) → dynamic target synchronized to anchor schedule.
- Conditioning vector: 6 values (V1) → 12 values (adds rise/fall time markers).

## 3. Physics-Forced Architectural Requirements

Derived from:
- `support/T2D.pdf`: Stokes-drag terminal-velocity derivation for cylindrical RecA-DNA, yielding `L(t) = C'·t^(2/3)` and `v(t) ∝ t^(-1/3)`.
- `support/FileFormat_probes.bin.V5.2.pdf`: file format spec (sample rate, molecule length ceiling, fold/structure extent fields).
- V1 rearchitecture memory (receptive field sizing for 4000-sample desert, bifurcation at 1x/2x for tag-width resolution).
- V2 goal 2 (fold segmentation as a new task).
- Deep Think review round 2 (bidirectional context requirement).

**An architecture MUST satisfy all 9:**

1. **Sample-level output resolution for velocity/heatmap heads.** Trailing-end 511bp tags are ~13.6 samples. Tag-width resolution is required on the heatmap so the post-peel L_heatmap_sharpness can drive concentration to single-sample-scale modes, and on the velocity head so Stokes drag profiles are representable without aliasing. Practical floor: 1x or 2x on the velocity head's output.
2. **Receptive field > 4000 samples somewhere in the network.** A missed tag merges two inter-probe gaps into ~20 kbp = ~100 ms = ~4000 samples at slow velocity. Network must reason across that gap.
3. **Conditioning on continuous physical observables.** 12-value FiLM/AdaLN vector (V1's 6 plus rise-time T10/T50/T90 and fall-time T90/T50/T10 from probes.bin molecule headers). No channel ID.
4. **Monotonic cumulative bp.** `cumulative_bp = cumsum(softplus(raw_velocity))`. Output-side, architecture-agnostic.
5. **Variable-length handling up to 80,000 samples, masked.** Per probes.bin spec, molecules can be up to 2 seconds at 25 μs sampling.
6. **Dense per-sample segmentation output.** Required by V2 goal 2 (fold/structure identification and clean-region gating).
7. **Parametric velocity prior support.** Must enable smooth-drag + residual-dip decomposition (explicit or implicit). V1's wide-kernel velocity branch does this implicitly; V2 retains that mechanism.
8. **No architectural barrier to training/inference on folded/structured molecules.** Implied by requirement 6 but called out separately.
9. **Strict bidirectional context (anti-causality).** Drag on any sample depends on the un-translocated molecule portion, which contains signal features that will translocate at *later* times. Global mixers in the network must attend forward and backward simultaneously. This rules out strictly-causal architectures (vanilla Mamba) unless they are bidirectional variants.

## 4. Scope Decisions

### 4.1 Preprocessing filter relaxation

**Current V1 behavior ([preprocess.py:142](src/mongoose/data/preprocess.py:142)):**
```python
if mol.structured or mol.folded_start or mol.folded_end or mol.do_not_use:
    continue
```
Structured, folded, and `do_not_use` molecules are dropped at cache time. They are absent from every existing cache.

**V2 behavior:** these four flags no longer cause exclusion. Molecules are retained with three new per-sample label arrays derived at preprocessing time:

1. **`seg_labels`** `[T]` int64 with 6 classes: `{clean=0, folded_start=1, folded_end=2, struct_recoverable=3, struct_amp_high=4, excluded_other=5}`. Densified from probes.bin's `folded_start_end_ms`, `folded_end_start_ms`, and per-structure `[start_ms, end_ms, attribute]` fields per Table 3/5 of the file-format spec.
2. **`accepted_peak_positions`** `[M]`: sample indices of probes with attribute bit 7 set and not excluded. Used as positive anchors during warmstart only.
3. **`rejected_peak_positions`** `[R]`: sample indices of probes with attribute bits 4/5/6/8 set. Used as negative anchors during warmstart only.

All wfmproc-derived supervision flows into the model through either the segmentation head (which inherits wfmproc's fold-detection judgment permanently) or the faded anchor term (which provides weak positional supervision during warmstart only). Post-peel, the model's peak detection is supervised exclusively by reference-alignment-based signals.

### 4.2 Segmentation head as first-class output

The new per-sample segmentation head is the architectural key that makes V2 goal 3 tractable. Without it, V2 would have to either (a) continue excluding folded molecules (failing goal 3) or (b) train peak detection on mixed clean/defect regions with no way to tell them apart (destroying the peak head's precision).

With the segmentation head:
- **Training:** peak-detection losses (L_bp, L_count, L_probe_anchor, L_heatmap_sharpness) are masked to `clean ∪ struct_recoverable` regions. L_segmentation itself is supervised everywhere, including defect regions, since defect classification is itself a training target.
- **Inference:** segmentation head argmax gates the peak list. Probes detected in `folded_start`, `folded_end`, `struct_amp_high`, or `excluded_other` regions are dropped. Each molecule's output carries a "quality annotation" (fraction of samples classified as clean) for downstream filtering.

### 4.3 Supervision approach: (B)-warmstart with soft anchors

V2 uses a two-phase supervision schedule on peak detection:

- **Warmstart (epochs 0 to `warmstart_epochs`):** accepted probe positions provide wide-boxcar positive anchors (amplitude 1.0 over ±500 samples); rejected probe positions provide narrow negative regions (±100 samples where heatmap mass is penalized). These are combined with a linearly fading blend factor that reaches 0 by epoch `warmstart_epochs`.
- **Post-peel (epoch ≥ `warmstart_epochs`):** peak detection is supervised exclusively by L_bp (bidirectional translation-invariant Sinkhorn-OT against reference bp), L_count (dynamic target), and L_heatmap_sharpness (negative-L2 concentration regularizer that stays on throughout). The velocity head is supervised only indirectly, via L_bp's constraint on cum_bp integrals between detected peaks.

This is categorically different from V1's `min_blend=0.1` hack, which kept the warmstart Gaussian term on forever. V2 fades the wfmproc-derived term to exactly zero on a defined schedule.

**Safety net:** L_heatmap_sharpness stays on through and past the peel transition. This prevents the known V1 heatmap-collapse failure mode.

## 5. Backbone: (H) 8x-Resolution Transformer U-Net

### 5.1 Why (H) won

Evaluated seven candidate families (A-G); detailed scoring table elided here (see review history). (H) was added mid-review and dominated on the following criteria:

1. **Full physics coverage (9/9).** Native sample-level output via CNN decoder. Native bidirectional context via self-attention. Preserves cumsum trick. FiLM + AdaLN carry conditioning.
2. **Fine-resolution global reasoning.** Transformer operates at 8x downsampled scale = 10,000 tokens max. Trailing-end tags are 1.7 tokens (Nyquist-adequate). Fold regions (multi-kbp, thousands of samples) are well-resolved at bottleneck and Transformer scales alike.
3. **80k-sample compute feasibility.** 10k tokens with FlashAttention-2 on A4500 is computationally comfortable; O(N) memory with sub-quadratic effective cost.
4. **Highest maturity among capable candidates.** Transformer blocks with RoPE, AdaLN, and FlashAttention are well-understood 2026 building blocks. No bespoke SSM implementation required; no variable-length-masking landmines.
5. **Clean path to SSL pretraining if V2.5 pursues it.** Transformer stack is substitutable with pretrained weights from a wav2vec2-style objective.

Alternatives considered and rejected:
- **(A) V1 as-is + segmentation head.** Dominated by (F) and (H) on long-range reasoning.
- **(F) V1 + Transformer *bottleneck* stack.** At 32x downsampling, tags are 0.425 tokens - below Nyquist. Bottleneck-only Transformer reasons over tag-blurred features; cannot do cross-tag morphological matching.
- **(G) CNN + Bi-Mamba.** Full physics coverage but variable-length masking trap in mamba-ssm's `selective_scan_fn` is a known failure mode; Bi-Mamba doubles SSM FLOPs; CNN-SSM impedance mismatch requires careful tuning.
- **(B) Conformer hybrid, (C) pure Transformer at k≤2, (D) vanilla causal Mamba, (E) wav2vec2-style.** Each eliminated on a specific capability, compute, or physics violation. Details in review history.

### 5.2 Tensor flow

```
input [B, 1, T]              T ≤ 80,000, variable per batch, padded + masked
  │
  ├─► Encoder (3-stage CNN, 8x cumulative downsampling, overlapping convs)
  │     Stage 0:  1x      [B,  64, T]
  │     Stage 1:  2x      [B, 128, T/2]
  │     Stage 2:  4x      [B, 256, T/4]
  │     Stage 3:  8x      [B, 512, T/8]
  │
  ├─► Transformer Stack (8 blocks, AdaLN + RoPE + FlashAttention)
  │     8x             [B, T/8, 512]
  │
  ├─► Decoder (2-stage CNN upsample, 8x→4x→2x, with skips)
  │     Stage U3:  4x     [B, 256, T/4]   + skip from enc Stage 2
  │     Stage U2:  2x     [B, 128, T/2]   + skip from enc Stage 1
  │
  ├─► BIFURCATION at 2x ────────┬─────────────────────────────┐
  │   Heatmap+Seg branch        │   Velocity branch           │
  │   U1H: 2x→1x + skip         │   U1V: 2x→1x + skip         │
  │   ResBlocks (k=7)           │   Wide-k ResBlocks (k=15)   │
  │   [B, 64, T]                │   [B, 64, T]                │
  │   Heatmap head (k=1) ► [B,T]│   Velocity head (k=1) ► v   │
  │   Seg head (k=1)    ► [B,K,T]  softplus(v) + ε            │
  │                             │   cumsum ► [B, T]           │
```

### 5.3 Encoder (3-stage overlapping-conv CNN)

Every downsample stage uses `kernel=7, stride=2` overlapping convolutions. No ViT-style non-overlapping patch strides. A 13.6-sample tag straddling a stride-2 boundary is still captured by neighboring receptive fields at each stage; by Stage 3 the tag's 1.7-token footprint has been smoothly pooled through three overlapping-window stages, not bucket-quantized.

```
Stage 0 (1x → 1x, raw-scale features):
  Conv1d(1, 64, kernel=15, padding=7, stride=1)      # wide initial smoothing
  FiLM(64, cond=metadata_embed_0)                    # V1 convention: raw physical scaling
  ResBlock(64, kernel=7) × 2

Stage 1 (1x → 2x):   Conv1d(64, 128, kernel=7, padding=3, stride=2);   ResBlock(128, kernel=7) × 2
Stage 2 (2x → 4x):   Conv1d(128, 256, kernel=7, padding=3, stride=2);  ResBlock(256, kernel=7) × 2
Stage 3 (4x → 8x):   Conv1d(256, 512, kernel=7, padding=3, stride=2);  ResBlock(512, kernel=7) × 2
```

ResBlock: `x + Conv1d(GELU(Conv1d(GroupNorm(x))))`. GroupNorm (not BatchNorm) due to small/variable batch composition.

### 5.4 Transformer stack (8 blocks at 8x resolution)

**Configuration:** `d_model=512, n_heads=8, d_head=64, d_ffn=2048, depth=8, dropout=0.1`.

Max tokens: 10,000 (80k / 8). Typical molecule: 375–2000 tokens. Memory at batch=8, max-length, bf16: ~3 GB activations across all 8 Transformer layers.

**Positional encoding (RoPE):** 1D Rotary Position Embeddings applied inside attention. θ_base = 50000 (higher than the default 10000 to stay accurate across the 10k-token context regime). No learned absolute embeddings anywhere.

```python
q_rot = rope(q, positions)   # rotation applied per-head before dot product
k_rot = rope(k, positions)
scores = q_rot @ k_rot.T
```

**Conditioning (AdaLN, DiT-style):** Metadata vector `m ∈ R^12` → `cond_embed = MLP(m) ∈ R^128`. Per block, project to six 512-dim vectors for `(shift_msa, scale_msa, gate_msa, shift_mlp, scale_mlp, gate_mlp)`. Zero-init the final projection's output weights; this makes each Transformer block the identity at initialization, allowing the model to train up capacity gradually.

```python
def block_forward(x, cond_embed, attn_mask, positions):
    s_a, c_a, g_a, s_f, c_f, g_f = project_cond(cond_embed)
    h = ln_attn(x) * (1 + c_a.unsqueeze(1)) + s_a.unsqueeze(1)
    h = attention(h, h, h, attn_mask=attn_mask, positions=positions)
    x = x + g_a.unsqueeze(1) * h
    h = ln_ffn(x) * (1 + c_f.unsqueeze(1)) + s_f.unsqueeze(1)
    h = ffn(h)
    x = x + g_f.unsqueeze(1) * h
    return x
```

**Attention kernel (FlashAttention-2 dispatch):** Every attention call goes through `F.scaled_dot_product_attention` with a boolean key-padding mask `[B, 1, 1, T/8]`. PyTorch 2.2+ dispatches to FlashAttention-2 when inputs are contiguous bf16/fp16, `is_causal=False`, and `attn_mask` is `None` or boolean.

```python
valid_mask_8x = downsample_mask(valid_mask_1x, factor=8)   # [B, T/8] bool
attn_mask = valid_mask_8x[:, None, None, :]                # [B, 1, 1, T/8]  True = attend

with torch.nn.attention.sdpa_kernel([SDPBackend.FLASH_ATTENTION]):  # dev: force dispatch
    y = F.scaled_dot_product_attention(
        q_rot, k_rot, v,
        attn_mask=attn_mask,
        dropout_p=0.1 if training else 0.0,
        is_causal=False,                   # bidirectional, satisfies physics req 9
    )
```

Dev/CI-time guard: wrap training step in `sdpa_kernel([FLASH_ATTENTION])` so PyTorch raises instead of silently falling back to the math backend on OOM.

### 5.5 Decoder (2-stage upsample + bifurcation)

```
Stage U3 (8x → 4x):
  PixelShuffle1d(scale=2)                    # implementation: see note below
  # enc Stage 3 output is 512 ch; PixelShuffle1d(2) reduces to 256 ch
  Concat skip from Encoder Stage 2 (256 ch) → 512 channels
  Conv1d(512, 256, k=1)                      # channel reduction after concat
  FiLM(256, cond=metadata_embed_bottleneck)  # V1 convention: global kinematics
  ResBlock(256, kernel=7) × 2

Stage U2 (4x → 2x):
  PixelShuffle1d(scale=2)                    # 256 ch → 128 ch
  Concat skip from Encoder Stage 1 (128 ch) → 256 channels
  Conv1d(256, 128, k=1)                      # channel reduction after concat
  ResBlock(128, kernel=7) × 2

[BIFURCATION at 2x — tensor copies into two branches]

Heatmap+Seg branch (U1H, 2x → 1x):
  PixelShuffle1d(scale=2)                    # 128 ch → 64 ch
  Concat skip from Encoder Stage 0 (64 ch)   → 128 channels
  Conv1d(128, 64, k=1)                       # channel reduction after concat (MUST INCLUDE)
  ResBlock(64, kernel=7) × 2
  → [B, 64, T]

Velocity branch (U1V, 2x → 1x, wide kernels):
  PixelShuffle1d(scale=2)                    # 128 ch → 64 ch
  Concat skip from Encoder Stage 0 (64 ch)   → 128 channels
  Conv1d(128, 64, k=1)                       # channel reduction after concat (MUST INCLUDE)
  WideResBlock(64, kernel=15) × 3            # learned low-pass; implicit parametric prior (req 7)
  → [B, 64, T]
```

**Implementation note on `PixelShuffle1d`.** PyTorch's `nn.PixelShuffle` is hardcoded for 4D spatial tensors `[B, C*r², H, W]` and will raise `RuntimeError` on a 3D `[B, C, T]` tensor. There is no built-in 1D variant. Two acceptable substitutes:

```python
# Option 1: einops (cleanest, recommended)
from einops import rearrange
x_up = rearrange(x, 'b (c r) t -> b c (t r)', r=2)   # r=upscale factor, c=out channels

# Option 2: manual view-based shuffle
b, cr, t = x.shape
c = cr // 2
x_up = x.view(b, c, 2, t).permute(0, 1, 3, 2).reshape(b, c, t * 2)

# Option 3: ConvTranspose1d (no channel reduction, so concat+reduce math changes)
x_up = nn.ConvTranspose1d(128, 128, kernel_size=4, stride=2, padding=1)(x)
# Then skip concat → 128+64=192 → Conv1d(192, 64, k=1)
```

The einops path is preferred: it is mathematically identical to `nn.PixelShuffle` and communicates intent clearly. Options 2 and 3 remain as fallbacks.

**Channel-reduction convs after every skip-concat are mandatory** — they are called out explicitly above in every decoder stage. Omitting them causes runtime channel-count mismatches at the first `ResBlock` after concat.

**Rationale for bifurcation at 2x:** the velocity branch needs a learned low-pass (wide kernels) to preserve the smooth Stokes drag profile while allowing local tag-induced dips. Bifurcating earlier (at 4x) would starve the heatmap branch of the encoder Stage 0 skip connection, which carries the sharpest features. 2x is the compromise.

### 5.6 Output heads

```
Heatmap:      Conv1d(64, 1, k=1) → logits [B, T]
              Training: BCE-with-logits
              Inference: sigmoid then NMS

### 5.7 Metadata conditioning pathway

```
metadata [B, 12] ─► MLP [Linear(12,128), SiLU, Linear(128,128), SiLU]
                       ↓
                   cond_embed [B, 128]
                       │
    ┌──────────────────┼──────────────────┬───────────────────────┐

FiLM at Enc Stage 0  FiLM at Dec U3    AdaLN in each Transformer  (skip-connection

### 5.8 Mask propagation

```
valid_mask_1x  [B, T]     bool, from collate
    │
    ├─ maxpool(k=2,s=2) ─► valid_mask_2x   → decoder U2 + U1H + U1V
    ├─ maxpool(k=2,s=2) ─► valid_mask_4x   → decoder U3
    └─ maxpool(k=2,s=2) ─► valid_mask_8x   → Transformer attn_mask
```

Max-pool (not min-pool) is conservative: a token is "valid" if any input sample in its 8-sample footprint is valid. Preserves gradient flow on partial-footprint tokens at molecule boundaries.

**Implementation note.** `F.max_pool1d` rejects `torch.bool` inputs and requires an explicit channel dimension. The correct pattern is:

```python
def _downsample_mask(valid_mask_1x: torch.Tensor) -> torch.Tensor:
    # valid_mask_1x: [B, T] bool
    m = valid_mask_1x.float().unsqueeze(1)                                # [B, 1, T]
    m = F.max_pool1d(m, kernel_size=2, stride=2)                          # [B, 1, T/2]
    return m.squeeze(1).bool()                                             # [B, T/2] bool
```

Apply three times in sequence to produce `valid_mask_2x`, `valid_mask_4x`, `valid_mask_8x`.

### 5.10 Known implementation gotchas

1. **RoPE dtype.** Apply rotation in fp32 then cast back to bf16 — pairwise sin/cos in bf16 can drift over 10k tokens.
2. **Mask dtype consistency.** `scaled_dot_product_attention` dispatches to FlashAttention only when mask is `torch.bool`. Float masks fall back to math backend. Assert in wrapper.
3. **AdaLN zero-init verification.** Assert `block.adaLN_modulation.weight.abs().max() < 1e-6` at construction. Non-zero init breaks identity-at-init.
4. **PixelShuffle1d channel constraint.** Upscale factor 2 requires channel count ÷ 2. Decoder heads use channel counts that satisfy this; verify in a tracer before first run.
5. **Cumsum numerical drift.** `torch.cumsum` in bf16 accumulates rounding error over 80k samples. Cast velocity to fp32 before cumsum (V1 already handles this).
6. **Encoder skip channel counts.** Stage 2 skip = 256 ch; decoder U3 after upsample = 256 ch; concat = 512 ch → reduce to 256 via 1x1 conv. Verify shapes.

## 6. Loss Composition

### 6.1 Loss-forced requirements

Derived from V2 goals, physics requirements, and architectural locks.

**Required loss terms (5 terms — L_velocity from V1 is removed, see section 6.4):**
1. **L_bp** — reference-aligned positional loss. Forced by V2 goal 1. Also the sole supervision path to the velocity head in V2.
2. **L_count** — total-probe-count regression. Forced by V2 goal 1.
3. **L_segmentation** — dense per-sample region classification. Forced by V2 goals 2, 3.
4. **L_probe_anchor** — faded warmstart positional scaffold. Forced by (B)-warmstart decision.
5. **L_heatmap_sharpness** — post-peel regularizer. Forced by heatmap-collapse risk at peel.

**Required properties (all terms):**
- Mask-aware (respect `[B, T]` padding).
- Clean-region gating for peak-detection losses (L_bp, L_count, L_probe_anchor, L_heatmap_sharpness). L_segmentation computes everywhere.
- Fully batched (no `for b in range(B)` python loops inside the hot path) — see Section 6.10#8 for the batching discipline.
- Lambda-scalable; scaling study required (V1's lambdas do not transfer).
- Directional invariance for L_bp (bidirectional Sinkhorn hedges aligner direction errors).

**Physics requirements NOT captured as loss terms:**
- Req 4 (monotonic bp): handled by `cumsum(softplus(v))` at output.
- Req 7 (parametric velocity prior): implicit via wide-kernel velocity branch + Sinkhorn integral constraints (see Section 6.4 for the rationale for dropping explicit L_velocity).
- Req 9 (bidirectional context): handled by attention architecture.

### 6.2 L_bp — Translation-Invariant Bidirectional Sinkhorn-OT, Batched

**Resolves three distinct V1-era problems:**

1. **The NMS autograd break.** V1's L_bp used hard-NMS to extract `peak_indices`, then computed `pred_bp_at_peaks = pred_bp_b[peak_indices]`. Gradient flowed through `pred_bp_b` (into velocity head) but NOT through `peak_indices` (into heatmap head). Post-peel, the heatmap would lose all positional gradient. Solution: treat the heatmap as a source probability distribution and compute Sinkhorn optimal transport against discrete reference probe locations in cumulative-bp coordinate space.

2. **The anchoring-bias landmine.** Naively anchoring predicted `cum_bp` at `bp[0]` (first active sample) forces Sinkhorn to minimize transport cost starting from whatever partial bp accumulation has happened before the first real probe. If there's dead buffer space at the molecule head (e.g., 5000 samples of signal before the first tag event), the model learns to predict near-zero velocity in that region to collapse the anchoring offset — violating the Stokes drag physics. Solution: compute the optimal-translation Wasserstein distance directly by mean-centering both distributions. `W_2²(μ, ν)_translation-free = W_2²(μ_centered, ν_centered)`. The loss measures only the relative spacing of peaks; any uniform shift is absorbed without gradient pressure on the velocity head.

3. **The per-batch-element dispatch bottleneck.** A Python `for b in range(B)` loop around a 50-iteration Sinkhorn means ~1600 sequential PyTorch kernel launches per step. At ~30μs kernel launch overhead, that's ~48 ms of pure CPU dispatch per step with GPU idle. Solution: batched Sinkhorn with `-inf` masking on padded regions, keeping all iterations as fused tensor ops over `[B, T, max_N]` shapes.

**Batched, translation-invariant formulation:**

```python
def _l_bp_batched(
    self,
    heatmap_probs: torch.Tensor,    # [B, T] sigmoid'd
    cum_bp:        torch.Tensor,    # [B, T]
    ref_padded:    torch.Tensor,    # [B, max_N]   padded from collate, temporal order
    ref_valid:     torch.Tensor,    # [B, max_N]   bool, True = valid entry
    active_mask:   torch.Tensor,    # [B, T]       bool, valid & clean
) -> torch.Tensor:
    """Bidirectional translation-invariant Sinkhorn-OT, fully batched.
    Returns [B] per-molecule losses; caller averages across batch."""
    device = heatmap_probs.device
    B, T = heatmap_probs.shape
    max_N = ref_padded.shape[1]
    if max_N < 2:
        return torch.zeros(B, device=device)

    N_valid = ref_valid.sum(dim=-1, keepdim=True).clamp(min=1.0).float()     # [B, 1]

    # Source: normalized heatmap mass over active samples
    source = heatmap_probs * active_mask.float()
    source = source / source.sum(dim=-1, keepdim=True).clamp(min=1e-8)       # [B, T]

    # --- Mass-weighted center of predicted distribution (common to both dirs) ---
    mu_pred = (source * cum_bp).sum(dim=-1, keepdim=True)                    # [B, 1]
    bp_centered = cum_bp - mu_pred                                            # [B, T]

    # --- Forward zero-anchored: anchor at first valid entry per row ---
    first_vals = ref_padded[:, :1]                                            # [B, 1]
    fwd_ref = (ref_padded - first_vals).abs()                                 # [B, max_N]

    # --- Reverse zero-anchored: anchor at last valid entry per row ---
    last_idx = (ref_valid.sum(dim=-1, keepdim=True) - 1).clamp(min=0).long()  # [B, 1]
    last_vals = ref_padded.gather(dim=-1, index=last_idx)                     # [B, 1]
    rev_ref = (ref_padded - last_vals).abs()                                  # [B, max_N]
    # NOTE: Sinkhorn is permutation-invariant on target; the SET of bp values
    # matters, not their order. So rev_ref doesn't need to be reversed in
    # entry order — the values are already correct for the reverse-direction
    # interval structure.

    span = fwd_ref.masked_fill(~ref_valid, 0.0).max(dim=-1, keepdim=True).values.clamp(min=1.0)  # [B, 1]

    L_fwd = self._sinkhorn_batched(bp_centered, fwd_ref, source, active_mask,
                                    ref_valid, span, N_valid)                 # [B]
    L_rev = self._sinkhorn_batched(bp_centered, rev_ref, source, active_mask,
                                    ref_valid, span, N_valid)                 # [B]

    tau = self.sinkhorn_soft_min_tau
    return -tau * torch.logsumexp(
        torch.stack([-L_fwd / tau, -L_rev / tau], dim=0), dim=0
    )                                                                          # [B]


def _sinkhorn_batched(
    self,
    bp_centered:  torch.Tensor,    # [B, T]      predicted cum_bp, already mean-centered
    ref_vals:     torch.Tensor,    # [B, max_N]  zero-anchored reference (fwd or rev)
    source:       torch.Tensor,    # [B, T]      normalized source distribution
    active_mask:  torch.Tensor,    # [B, T]
    ref_valid:    torch.Tensor,    # [B, max_N]
    span:         torch.Tensor,    # [B, 1]
    N_valid:      torch.Tensor,    # [B, 1]
) -> torch.Tensor:
    """One Sinkhorn direction. Returns [B] per-molecule losses."""
    # Mean-center the reference (translation invariance completed)
    mu_ref = (ref_vals * ref_valid.float()).sum(dim=-1, keepdim=True) / N_valid   # [B, 1]
    ref_centered = (ref_vals - mu_ref) * ref_valid.float()                        # [B, max_N]

    # Translation-invariant cost matrix: [B, T, max_N], normalized by span²
    cost = (
        (bp_centered.unsqueeze(-1) - ref_centered.unsqueeze(1)) ** 2
    ) / (span.unsqueeze(-1) ** 2)

    valid_pair = active_mask.unsqueeze(-1) & ref_valid.unsqueeze(1)               # [B, T, max_N]
    log_K = (-cost / self.sinkhorn_eps).masked_fill(~valid_pair, float('-inf'))

    log_s = torch.log(source.clamp(min=1e-8)).masked_fill(~active_mask, float('-inf'))    # [B, T]
    log_t = (-torch.log(N_valid)).expand(-1, ref_valid.shape[1]).masked_fill(
        ~ref_valid, float('-inf'))                                                  # [B, max_N]

    log_u = log_s.clone()
    log_v = log_t.clone()
    # NaN-safe logsumexp: padded rows are entirely -inf, and logsumexp([-inf,...,])
    # returns -inf whose gradient is softmax([-inf,...,]) = NaN. Any downstream
    # 0.0 * NaN = NaN, poisoning weights on the first batch. Neutralize padded
    # rows to 0 before the reduction, then restore the -inf mask on the output.
    for _ in range(self.sinkhorn_iters):
        # log_u update: reduce over ref dimension (dim=-1); rows without any
        # valid key (a padded source sample) must be neutralized.
        log_Ku = torch.where(active_mask.unsqueeze(-1),
                             log_K + log_v.unsqueeze(1),
                             torch.zeros_like(log_K))
        lse_u = torch.logsumexp(log_Ku, dim=-1)                                   # [B, T]
        log_u = torch.where(active_mask, log_s - lse_u, torch.full_like(log_s, float('-inf')))

        # log_v update: reduce over source dimension (dim=1); columns without
        # any valid query (a padded target entry) must be neutralized.
        log_Kv = torch.where(ref_valid.unsqueeze(1),
                             log_K + log_u.unsqueeze(-1),
                             torch.zeros_like(log_K))
        lse_v = torch.logsumexp(log_Kv, dim=1)                                    # [B, max_N]
        log_v = torch.where(ref_valid, log_t - lse_v, torch.full_like(log_t, float('-inf')))

    log_P = log_K + log_u.unsqueeze(-1) + log_v.unsqueeze(1)                      # [B, T, max_N]
    P_cost = (log_P.exp() * cost).masked_fill(~valid_pair, 0.0)
    return P_cost.sum(dim=(-2, -1)) * (span.squeeze(-1) ** 2)                     # [B]
```

**Collate responsibility (new):** the DataLoader collate_fn must produce `ref_padded: [B, max_N]` and `ref_valid: [B, max_N] bool` tensors in place of V1's `reference_bp_positions_list: list[Tensor]`. This is a one-line change in the collate: pad the per-molecule ref tensors to the batch max, emit a validity mask alongside.

**Alternative implementation path: `geomloss.SamplesLoss`.** Jean Feydy's `geomloss` library provides batched Sinkhorn with KeOps-optimized CUDA kernels that natively handle variable-length point clouds. If available in the environment, prefer `geomloss.SamplesLoss(loss='sinkhorn', blur=eps, scaling=0.5)` over the hand-rolled implementation — faster, better-debugged, and handles numerical edge cases that the inline version doesn't. Hand-rolled version above is the reference semantics; geomloss is the production recommendation when its KeOps dependency is acceptable.

**Gradient paths:**
- To **heatmap logits** via the source distribution (the critical path that fixes the NMS break).
- To **velocity** via `cum_bp` appearing in both `mu_pred` and `bp_centered` → into the cost matrix.
- To **segmentation head**: none. `active_mask` is derived from hard-argmaxed seg labels and used as a gate only; training of the segmentation head comes exclusively from L_segmentation.

### 6.3 L_count — dynamic target synchronized to anchor schedule, batched

**The V1 Gaussian area trap, resolved.** V1 targeted `sum(heatmap) = n_ref_probes` regardless of phase. During warmstart, wide anchor targets have per-peak integral >> 1, causing L_count to fight L_probe_anchor. Solution: during warmstart, target is `sum(target_anchor_heatmap)`; post-peel, target is `n_ref_probes`. Transition matches the anchor peel schedule.

```python
def _l_count_batched(self,
                     heatmap_probs:       torch.Tensor,   # [B, T]
                     target_anchor_mass:  torch.Tensor,   # [B] — precomputed sum; None post-peel
                     n_ref_probes:        torch.Tensor,   # [B] int — MUST be batched
                     active_mask:         torch.Tensor,   # [B, T] bool
                    ) -> torch.Tensor:
    heatmap_sum = (heatmap_probs * active_mask.float()).sum(dim=-1)              # [B]
    if target_anchor_mass is not None and self._blend > 0:
        # target_anchor_mass is the anchor-heatmap integral computed once per
        # batch in _l_probe_anchor_batched and passed through.
        target = (self._blend * target_anchor_mass.detach()
                  + (1.0 - self._blend) * n_ref_probes.float())                  # [B]
    else:
        target = n_ref_probes.float()                                             # [B]
    return F.smooth_l1_loss(heatmap_sum, target, reduction='mean')
```

**Batching correctness.** `heatmap_sum`, `target_anchor_mass`, and `n_ref_probes` are all `[B]` tensors; `smooth_l1_loss` computes an element-wise error and averages. This replaces V1's scalar-target formulation which would have broadcast incorrectly once the outer loop was removed.

### 6.4 L_velocity — REMOVED IN V2

**V1 had an explicit L_velocity loss:** MSE between the velocity head's output at NMS-detected peak samples and a physics target `511 bp / FWHM_ms`, with FWHM measured from the heatmap itself.

**V2 drops this loss entirely.** The formulation has a fatal internal contradiction once V2 decouples the heatmap from V1's Gaussian-target supervision:

- **During warmstart** (epochs 0–2, blend=1.0), L_probe_anchor supervises the heatmap toward a ~1000-sample flat boxcar. Measured FWHM is ~1000 samples. Target velocity becomes `511 bp / (1000 × 0.025 ms) ≈ 20 bp/ms` — essentially 0.5 bp/sample, orders of magnitude below physical terminal velocity. L_velocity would actively pull the velocity head's output toward this incorrect target, fighting L_bp's scaling pressure.
- **Post-peel** (epoch ≥5, blend=0), L_heatmap_sharpness drives the heatmap toward a Dirac spike per detected peak. Measured FWHM approaches 1 sample. Target velocity blows up to `511 bp / (0.025 ms) = 20440 bp/ms`, roughly 10× the fastest physically observed molecule velocity. L_velocity's gradients to the velocity head would explode.

**The contradiction is structural**: V1's L_velocity worked because V1's focal loss pinned the heatmap's width to actual tag durations. V2 deliberately decouples heatmap shape from waveform-level tag width (the heatmap now represents peak *probability*, not tag morphology). That decoupling is by design — but it invalidates the FWHM-based velocity measurement.

**Why velocity supervision is still adequate without an explicit term:**

1. **Sinkhorn integral constraints.** Translation-invariant Sinkhorn-OT pins the *relative* spacing of cum_bp at detected peaks to the reference bp intervals. Between consecutive peaks `i` and `j`, `cum_bp[t_j] − cum_bp[t_i] ≈ ref_bp[j] − ref_bp[i]`. This constrains the integral of velocity over each inter-peak segment to the correct bp distance — a strong physics-aware constraint.
2. **Architectural smoothness prior.** The wide-kernel velocity branch (WideResBlock with kernel=15) is a learned low-pass filter; the velocity output cannot contain frequencies sharper than the Stokes drag profile's natural bandwidth.
3. **Conditioning on kinematic observables.** The 12-value FiLM/AdaLN vector (baseline, duration, rise/fall times) encodes per-molecule velocity priors directly into the backbone and decoder.

**Trade-off acknowledgement.** Without explicit velocity supervision, the model's velocity output *shape* (the specific power-law curve) is only indirectly supervised. Section 8's power-law-α diagnostic is the watch signal: if post-training α is out of the `[0.4, 0.7]` empirical range, or if the `FWHM × v_at_peak` sanity check deviates from 511 bp, we promote back to an explicit velocity loss in V2.1 — but formulated against the input waveform or encoder features, not against the heatmap's FWHM.

### 6.5 L_segmentation — multi-class CE with class weights

```python
def _l_segmentation(self, seg_logits, seg_labels, valid_mask):
    loss_per_sample = F.cross_entropy(
        seg_logits, seg_labels,
        weight=self.seg_class_weights,   # [K] from cache statistics
        reduction='none',
    )                                    # [B, T]
    return (loss_per_sample * valid_mask.float()).sum() / valid_mask.float().sum().clamp(min=1.0)
```

**Class weights** computed as `sqrt(total / (K * count_k))` per class, computed once from the full cache at preprocessing time and persisted. Without weighting, the model trivially predicts "clean" everywhere and achieves high accuracy while missing every fold.

### 6.6 L_probe_anchor — wide boxcar + narrow negatives, faded

```python
def _build_anchor_heatmap(self, T, accepted_pos, rejected_pos, device):
    target_pos = torch.zeros(T, device=device)
    target_neg = torch.zeros(T, device=device, dtype=torch.bool)
    Wp = self.anchor_pos_window_samples   # 500
    Wn = self.anchor_neg_window_samples   # 100
    for p in accepted_pos.tolist():
        lo, hi = max(0, p - Wp), min(T, p + Wp + 1)
        target_pos[lo:hi] = 1.0           # un-normalized boxcar, amplitude 1.0
    for p in rejected_pos.tolist():
        lo, hi = max(0, p - Wn), min(T, p + Wn + 1)
        target_neg[lo:hi] = True
    return target_pos, target_neg

def _l_probe_anchor(self, heatmap_probs, heatmap_logits, accepted_pos, rejected_pos, active_mask):
    if self._blend <= 0.0:
        return torch.zeros((), device=heatmap_probs.device), None

    T = heatmap_probs.shape[0]
    target_pos, target_neg_mask = self._build_anchor_heatmap(T, accepted_pos, rejected_pos,
                                                              heatmap_probs.device)
    target_pos = target_pos * active_mask.float()
    neg_mask = target_neg_mask & active_mask

    L_pos = F.binary_cross_entropy_with_logits(
        heatmap_logits, target_pos, reduction='none'
    ) * active_mask.float()
    L_pos = L_pos.sum() / active_mask.float().sum().clamp(min=1.0)

    L_neg = (heatmap_probs * neg_mask.float()).sum() / neg_mask.float().sum().clamp(min=1.0)

    return self._blend * (L_pos + L_neg), target_pos   # target returned for L_count
```

**Why BCE on a 0/1 boxcar, not focal on Gaussians:** focal's class-imbalance weighting is designed for sparse-positive labels. A 1000-sample boxcar window contains ~1000 "positive" samples per probe; focal would severely under-weight them. Plain BCE handles this correctly. The model is supervised to produce ≥0.5 probability *somewhere* inside each window, not a specific peak shape.

### 6.7 L_heatmap_sharpness — negative L2 norm (concentration regularizer)

**V1-style formulation would have collapsed training.** A naive `(max_pool(m) − m)²` objective is minimized when `max_pool(m) = m` everywhere — which happens when `m` is *flat*. The regularizer as originally proposed penalizes sharpness and rewards uniform smearing; post-peel the model would instantly flatten its logits to drive this term to zero, destroying peak detection.

**Correct formulation: maximize L2 norm under the L_count L1 constraint.** For a vector with fixed `‖m‖₁` (enforced by L_count), `‖m‖₂` is maximized when mass concentrates into a few samples and minimized when it spreads uniformly. Minimizing the negative L2 norm therefore encourages concentration directly.

```python
def _l_sharpness_batched(self,
                         heatmap_probs: torch.Tensor,    # [B, T]
                         active_mask:   torch.Tensor,    # [B, T] bool
                        ) -> torch.Tensor:
    m = heatmap_probs * active_mask.float()                                      # [B, T]
    # L2² per molecule, normalized by active length for molecule-size invariance
    l2_sq = (m ** 2).sum(dim=-1)                                                  # [B]
    n_active = active_mask.float().sum(dim=-1).clamp(min=1.0)                     # [B]
    return -(l2_sq / n_active).mean()    # minimize negative L2 = maximize L2
```

**Interaction with L_count (the constraint):** L_count enforces `‖m · active_mask‖₁ ≈ n_ref_probes` (or the dynamic anchor target during warmstart). Under that L1 constraint, the L2-maximizing distribution is `n_ref_probes` delta spikes each with probability 1.0 — i.e., perfectly-peaky heatmap. In practice SGD produces narrow-but-not-degenerate peaks, which is the desired behavior.

**Why this works without causing the exploding L_velocity gradients described in 6.4:** L_velocity is removed. The Dirac-limit heatmap is the correct target shape once velocity has no FWHM-based supervision.

**Alternative formulation (entropy minimization).** Shannon entropy `−∑ p log p` is also minimized by a concentrated distribution. Mathematically equivalent asymptotically; the negative-L2 form is cheaper and has stabler gradients at near-zero probabilities. Keep L2 unless a specific training pathology appears.

### 6.8 Warmstart-then-peel schedule

```python
def set_epoch(self, epoch):
    W = self.warmstart_epochs       # default 5
    F = self.warmstart_fade_epochs  # default 3
    full = max(W - F, 0)
    if W <= 0 or epoch >= W:
        self._blend = 0.0
    elif epoch < full:
        self._blend = 1.0
    else:
        frac = (epoch - full + 1) / max(F, 1)
        self._blend = max(0.0, 1.0 - frac)
```

Default schedule: full warmstart for 2 epochs, linear fade over 3 epochs, zero from epoch 5 onward. Tunable.

### 6.10 Known loss-level subtleties

1. **Sinkhorn `eps` tuning.** Too small = explosive gradients on near-degenerate transport plans; too large = blurry transport = weak positional signal. Start `eps=1.0` (cost normalized to unit span²). Sweep {0.1, 0.5, 1.0, 2.0} on a short training run before committing.
2. **Sinkhorn iteration count.** 50 is conservative; 20 often suffices once `eps` is tuned. Profile and reduce if tolerances allow. geomloss typically converges in 15–30 scaled iterations at default blur.
3. **Sinkhorn NaN trap on fully-padded rows.** The Sinkhorn implementation in Section 6.2 uses `torch.where(..., log_K, 0.0)` to neutralize fully-padded rows/columns before `logsumexp`. Naive `-inf` masking produces `logsumexp([-inf, -inf, ...]) = -inf`, whose gradient is `softmax([-inf, -inf, ...]) = [NaN, NaN, ...]`. Any downstream `0.0 * NaN` computation (e.g., the `masked_fill(~valid_pair, 0.0)` on the transport cost) then produces NaN on the first backward pass, permanently poisoning weights. **The `torch.where` pattern is mandatory, not optional.** Confirm by running A2a's gradient sanity test with an intentionally fully-padded batch element — V2 must still produce finite gradients on the other batch members.
4. **Collate changes required for L_bp.** The batched Sinkhorn needs `ref_padded [B, max_N]` and `ref_valid [B, max_N]` tensors from the DataLoader, replacing V1's list-of-Tensor output. Update `collate.py` accordingly. If using geomloss, pass variable-length point clouds natively via its `x` and `y` arguments.
5. **Cost-matrix memory at batch scale.** Batched cost is `[B, T, max_N]`. For B=8, max T=80k, max_N=40: 8 × 80k × 40 × 4 bytes (fp32) = ~100 MB per direction, ~200 MB for both. Fine on A4500 24 GB. If tight, either stratified-subsample the source distribution to `T_sub ≤ 20k` samples per molecule, or switch to geomloss (which uses KeOps lazy tensors and avoids materializing the full matrix).
6. **First-order gradient sanity test (before any training).** Feed a minibatch at each phase (warmstart, mid-fade, post-peel), call `total.backward()`, and assert `pred_heatmap_logits.grad`, `raw_velocity.grad`, `pred_seg_logits.grad` are all non-zero AND finite (no NaN). The post-peel heatmap grad is the specific test that would have caught the original NMS break; the NaN check is what catches the Sinkhorn-fully-padded-row trap.
7. **Translation-invariance edge case: mass-weighted mean on nearly-empty heatmaps.** Very early in training, the heatmap may be near-uniform. `mu_pred = (source * cum_bp).sum()` is then close to the midpoint of `cum_bp` over active samples — a reasonable center regardless. No special handling needed, but monitor `mu_pred`'s variance across the batch in the first few epochs.
8. **Full batching discipline — no `for b in range(B)` loops in the loss path.** Every loss term in this section operates on `[B, ...]` tensors end-to-end: L_bp via batched Sinkhorn, L_count via per-molecule sums reduced to `[B]`, L_segmentation via masked cross-entropy, L_probe_anchor via per-sample boxcar construction (vectorizable with `torch.where` against precomputed per-molecule window masks from the collate), L_heatmap_sharpness via per-molecule L2-norm reduction. The per-molecule python loop in V1's `combined.py` should not appear anywhere in V2's `CombinedLossV2.__call__`.
9. **Class-weight skew in L_segmentation.** If cache statistics show `excluded_other` is very rare (say <0.1% of samples), its `sqrt(total / (K * count_k))` weight will be ≥30× clean's weight, causing gradient spikes at rare boundary samples. Cap class weights at 10× clean baseline.

## 7. Training Curriculum

### 7.1 Decision: no temporal data-filtering curriculum

V2 trains on the full cached dataset from epoch 0. There is no phase-based or quality-quartile data-filtering curriculum. The only temporal schedule in V2 training is the warmstart-then-peel blend factor on L_probe_anchor (section 6.8) and standard LR scheduling.

### 7.2 Justification

Three reasons the obvious alternative (phase-based progressive introduction of structured/folded molecules) is rejected:

1. **Peak heads are structurally insulated from defect regions by GT-derived masking.** The `active_mask` used to gate L_bp, L_count, L_probe_anchor, and L_heatmap_sharpness is derived from the ground-truth `seg_labels` tensor (`clean ∪ struct_recoverable`), not from the segmentation head's predictions. Peak-detection losses therefore see only clean-region samples from day 0, regardless of what the model thinks about fold boundaries, and regardless of which molecules are in the current batch. The "fold contamination of peak training" concern that would motivate a data curriculum doesn't exist under this architecture.

2. **Shared-backbone representation shift at phase boundaries.** Under a phase-based curriculum, during Phase 1 (easy molecules only), the shared 8x-Res Transformer backbone would train its representations purely on easy-molecule signal statistics. When Phase 2 introduces structured/folded molecules, the backbone faces a distribution shift and must simultaneously (a) learn the new input distribution and (b) preserve peak-detection representations that were stable under the narrower Phase 1 distribution. Since the backbone is shared between segmentation and peak heads, this is a known curriculum-learning failure mode (catastrophic interference at phase boundaries) and the coupling makes it more acute than in isolated-head settings. Under Option (D), the backbone sees the full distribution from day 0 and converges to a single stable representation.

3. **The warmstart-then-peel schedule is already a temporal curriculum in disguise.** Epochs 0–2 have full wfmproc-derived anchor supervision; epochs 2–5 linearly fade it out. This IS a curriculum — a temporal progression from supervised-with-scaffolding to self-supervised. Adding a second temporal axis (data-filtering) compounds schedule-tuning risk without a clear capability gain.

### 7.3 Static quality floor (permanent preprocessing filter, not a curriculum)

A fixed quality filter is applied at preprocessing time to protect the loss pipeline from degenerate alignment cases:

```
match_rate       = n_ref_probes / max(n_wfmproc_probes, 1)
accepted         = (match_rate >= MATCH_RATE_FLOOR) AND (n_ref_probes >= MIN_REF_PROBES)
```

Molecules failing these checks are excluded from the cache entirely.

**Why this is NOT a curriculum:** applied once at preprocessing, not progressively over epochs. Reflects static data quality, not difficulty progression. Primary purpose is numerical stability of Sinkhorn-OT — a molecule with 3 matched probes out of 20 wfmproc probes gives a meaningless alignment signal that injects noisy gradients into the shared heatmap head.

**Starting thresholds (tunable):**
- `MATCH_RATE_FLOOR = 0.5` — require at least half of wfmproc-detected probes to have reference-matched counterparts.
- `MIN_REF_PROBES = 10` — inherits V1's `min_probes` convention.

### 7.4 Summary: V2 training schedule

**Temporal dimensions of V2 training:**
- Learning rate schedule (linear warmup → cosine decay; standard recipe).
- L_probe_anchor blend factor: 1.0 for epochs 0–2, linearly fading to 0 by epoch 5, 0 thereafter.
- L_count target: dynamically tracking L_probe_anchor blend per section 6.3.
- Everything else constant from epoch 0 to completion.

**Data dimensions (all static, set at preprocessing):**
- Match rate floor: 0.5.
- Minimum reference probes: 10.
- All other molecules — including `structured`, `folded_start`, `folded_end`, and `do_not_use` — retained with densified per-sample segmentation labels.

## 8. Evaluation Harness

### 8.1 Goals

The V2 evaluation harness must:
1. Measure V2 against V1 rearchitecture on reference-bp accuracy — the metric most directly tied to the product value.
2. Stratify results across dimensions V2 explicitly claims to improve (structured/folded molecules, long molecules, low-match-rate molecules).
3. Measure the segmentation head independently — per-class F1 and boundary localization.
4. Sanity-check the velocity head against the Stokes power-law prior (physics requirement 7).

### 8.2 Primary metrics

All computed after applying the segmentation head's predicted clean-region gate at inference (not GT gate — this is the "real" deployment conditional).

**Peak-detection quality (reference-bp space):**
- **Peak F1 at tolerance ε.** Hungarian 1:1 matching between predicted peaks and reference probe bp positions; a match farther than ε bp counts as a miss. Report at ε ∈ {500, 1000, 2000} bp.
- **Bp-MAE per molecule.** Mean absolute reference-bp error across matched peak pairs in one molecule.
- **Bp-MAE aggregate.** Median and mean of per-molecule bp-MAE across held-out molecules.
- **Precision, recall** separately — F1 alone hides asymmetric failure modes (over-calling vs under-calling).
- **Peak-count correlation.** Pearson correlation of predicted peak count against reference probe count, per molecule.

**Segmentation quality:**
- **Per-class F1** on the 6 segmentation classes.
- **Confusion matrix** (6 × 6).
- **Boundary localization accuracy.** Fraction of fold/structure boundary transitions localized within ±N samples, N ∈ {50, 200, 1000}.

**Velocity-head sanity (non-metric diagnostics):**
- **Power-law fit exponent α** for predicted velocity on held-out clean molecules. Fit `v(t) ∝ t^α` on the smoothed (low-passed) velocity output; α should be in [0.4, 0.7], consistent with the empirical T2D fits recorded in the ground-truth-strategy memory (α ∈ [0.543, 0.577]).
- **Predicted vs TDB edge velocity correlation.** Predicted velocity at the molecule's rising edge should correlate with the TDB-derived rise-time velocity from the `rise_time_T10/T50/T90` markers.
- **FWHM × v_at_peak ≈ 511 bp.** At each detected peak, the product of the heatmap FWHM (ms) and the predicted velocity at that peak (bp/ms) should be close to 511. Deviation indicates a mismatch between the peak head and the velocity head.

### 8.3 Stratification dimensions

Every peak-detection metric is reported both aggregate and stratified across:

1. **Segmentation-class profile (per molecule).** `all_clean` / `has_folded_end` / `has_folded_start` / `has_struct_recoverable` / `has_struct_amp_high` / `mixed_defect`. This is the key V2-goal-3 stratification.
2. **Match-rate quartile** at preprocessing (recorded per molecule, not used for training filtering beyond the static floor).
3. **Molecule length quartile** by `transloc_time_ms`. Long molecules stress the 4000-sample-desert case.
4. **Num-probes bucket** by `n_ref_probes`: {10–20, 20–40, 40+}.
5. **Direction** (forward vs reverse). Bidirectional Sinkhorn soft-min should produce symmetric results — this stratification is a sanity check on the translation-invariance fix.

### 8.4 Structured-molecule rescue rate (the V2-goal-3 headline metric)

**Definition.** A molecule is "rescued by V2" if:
- It has at least one non-clean segmentation class present (i.e., `structured`, `folded_start`, or `folded_end` flag set in probes.bin), AND
- V2's peak F1 at tolerance 1000 bp on this molecule is ≥ 0.5, AND
- V1 rearchitecture's peak F1 at the same tolerance on this molecule is < 0.5 (V1 failed).

**Rescue rate = fraction of structurally-defective held-out molecules satisfying all three conditions.**

V2 goal 3 requires `rescue_rate > 0` demonstrably. Larger rescue rates indicate stronger V2 value delta over V1.

### 8.5 Held-out split strategy

Two orthogonal splits:
1. **Run-level holdout.** Hold out 2–3 complete E. coli runs from training. Tests cross-channel and cross-condition generalization — most relevant to eventual human-sample deployment.
2. **Within-run holdout.** Within each training run, hold out 10% of molecules (seeded). Tests intra-run generalization, cheaper to iterate on during development.

All metrics reported separately on both splits. Training-time monitoring uses the within-run holdout; final V2-vs-V1 numbers are reported on the run-level holdout.

### 8.6 Baselines

Every V2 evaluation run compares against:
1. **V1 rearchitecture** (current sprint branch `claude/peaceful-rubin-bfb7a9`, Micro Phase 6 F1 = 0.917 on wfmproc replication). Run V1 inference on the same held-out molecules; compute identical reference-bp metrics.
2. **wfmproc / legacy T2D.** Treat the existing `probes.bin`'s accepted-probe positions as a baseline peak list; convert to reference bp via `reference_map.bin`. This is the system V2 is ultimately replacing.

Primary table per eval run:

## 9. Ablation Plan

### 9.1 Strategy

Each ablation asks a specific falsifiable question about one V2 component. Primary reported metric is Peak F1 @ 1000 bp on the structured-molecule holdout (the subset V2 goal 3 targets). Secondary metrics (bp-MAE, seg F1, rescue rate) are reported when they matter for interpretation.

### 9.2 Primary hypotheses — each has a targeted ablation

#### H1: The 8x-Res Transformer backbone > V1 CNN U-Net for fold-aware peak detection

**Ablation A1.** Train a V2-minus-Transformer model: keep all V2 losses, the segmentation head, and the relaxed preprocessing, but replace the 8x-Res Transformer backbone with V1 rearchitecture's pure CNN U-Net (5-level encoder, single-MHSA bottleneck). All other hyperparameters constant.

- **Falsifies H1 if:** V2-minus-Transformer's Peak F1 @ 1000 bp on structured molecules matches V2's within 1.0% absolute.
- **Confirms H1 if:** V2 exceeds V2-minus-Transformer by ≥ 3% absolute on that metric.

#### H2: Bidirectional Sinkhorn-OT fixes the NMS gradient break for post-peel heatmap training

**Ablation A2a (static gradient-flow test, free).** Before any training, run the first-order gradient sanity test from Section 6.10#5 on both V2 and V2-minus-Sinkhorn (where L_bp reverts to V1's hard-NMS + soft-DTW). At each training phase (warmstart, mid-fade, post-peel), confirm V2 produces non-zero `pred_heatmap_logits.grad` and V2-minus-Sinkhorn does not at the post-peel phase. This ablation requires zero training.

**Ablation A2b (end-to-end).** Train V2-minus-Sinkhorn to completion. Track heatmap peakiness (value of `peakiness_regularizer` on held-out) across epochs. Expectation: degradation or drift after the peel-off epoch (~5) as the heatmap loses its positional gradient.

- **Falsifies H2 if:** V2-minus-Sinkhorn's heatmap does not drift post-peel and its F1 matches V2's.
- **Confirms H2 if:** V2-minus-Sinkhorn diverges post-peel (heatmap spreads, F1 drops) while V2 does not.

#### H3: The segmentation head enables structured/folded-molecule peak rescue

**Ablation A3.** Train V2-minus-Segmentation: remove the segmentation head and L_segmentation term. Replace the GT-derived `clean_mask` with an all-ones mask (peak losses apply across the entire molecule including defect regions). Compute rescue rate on held-out structured molecules.

- **Falsifies H3 if:** V2-minus-Segmentation rescues structured molecules at the same rate as V2.
- **Confirms H3 if:** V2 rescue rate > 2× V2-minus-Segmentation's.

#### H4: Warmstart-then-peel with wide-boxcar anchors avoids post-peel heatmap collapse

**Ablation A4a (no warmstart).** Train V2 with `blend=0` from epoch 0. L_probe_anchor never contributes. Measures whether training can converge at all without wfmproc-derived positional scaffolding.

**Ablation A4b (no peel — "V1 Min-Blend hack").** Train V2 with `blend=1` forever. Model never leaves the wfmproc supervision regime. Measures how much of V1's reported F1 = 0.917 is wfmproc-recapitulation vs. true reference-bp accuracy.

- **Falsifies H4 if:** V2-no-warmstart converges to the same F1 as V2. (Would mean warmstart is unnecessary.)
- **Diagnostic (not falsification) result:** V2-no-peel's F1 characterizes how much V2's true accuracy comes from the peel itself.

### 9.3 Secondary ablations — component-level refinement

**A5: Translation-invariant vs naive anchoring in Sinkhorn.** Replace `bp_centered = bp - mu_pred` with `bp_centered = bp - bp[0]`. Expect: V2-naive-anchor underperforms on molecules with long buffer regions before the first probe (where the anchoring-bias landmine bites). Look for degradation stratified by "leading buffer length" — samples before the first heatmap peak.

**A6: Wide-boxcar anchor vs narrow Gaussian anchor (V1 warmstart style).** Replace L_probe_anchor's boxcar positive with V1-style narrow Gaussian targets + focal loss. Expect: heatmap amplitude oscillation during warmstart from the Gaussian-Area Trap (even with the dynamic L_count target, since anchor area remains structurally different).

**A7: Dynamic L_count target vs static.** Keep L_count targeting `n_ref_probes` throughout, regardless of phase. Expect: heatmap amplitude oscillation during warmstart (original V1 Ghost).

**A8: AdaLN in Transformer vs FiLM-everywhere.** Replace Transformer AdaLN with FiLM layers. Expect: minor metric regression but noticeably less stable early training (per DiT paper's zero-init stability argument).

**A9: RoPE vs learned absolute positional embeddings.** Expect: worse length generalization on held-out molecules longer than any seen in training. Specifically, degradation of `Peak F1` as a function of molecule length on the length-quartile stratification.

**A10: Overlapping convs vs non-overlapping patches in encoder.** Replace `kernel=7, stride=2` with `kernel=2, stride=2` non-overlapping. Expect: aliasing of trailing-end tags (13.6-sample tags at patch-boundary positions get dropped). Measurable degradation on Peak F1 for molecules with many trailing-end probes.

**A11: 12-value FiLM vs 6-value (V1 style).** Drop the 6 rise/fall time markers from the conditioning vector. Expect: small degradation on length-outlier and velocity-outlier molecules, minimal on typical molecules.

**A12: Folded/structured inclusion in training vs V1 preprocessing filter.** Train V2 with V1's preprocessing filter applied (drop all `structured | folded_* | do_not_use`), then evaluate on the full eval set. Expect: V2-V1-filter lacks structured-molecule rescue capability entirely. This isolates the benefit of V2's data expansion from the benefit of its architecture — important for attributing which change matters more.

### 9.4 Priority and dependencies

**Tier 1 — must run to validate V2's primary claims:** A1, A2a (free), A3, A4a.

**Tier 2 — run if compute permits; high diagnostic value:** A2b, A4b, A5, A12.

**Tier 3 — design refinement; defer if Tier 1/2 already validated the architecture:** A6, A7, A8, A9, A10, A11.

**Dependencies:**
- A2a can run before any training (it's a gradient-sanity check).
- A1 and A3 are the two largest compute sinks (full V2-style training with a modified component); run in parallel if compute allows.
- Other Tier 1 ablations can be run serially on released compute.

### 9.5 Reporting

Each ablation produces:
- **Primary:** Reference-bp Peak F1 @ 1000 bp on structured-molecule holdout.
- **Secondary:** Peak F1 on all-molecule holdout, rescue rate, per-class seg F1.
- **Diagnostic:** Loss-component trajectories during training (where did the ablated model go wrong?).
- **Special for A2a and A2b:** Gradient-flow snapshots at epoch 0, mid-warmstart, and post-peel.

- **Deep Think review 2** (2026-04-20): validated blueprint for (H). Injected four implementation mandates: RoPE positional encoding, overlapping-conv anti-aliasing, AdaLN conditioning, FlashAttention dispatch with boolean masks.
- **Deep Think review 3** (2026-04-20): on initial loss composition. Identified the NMS autograd break in L_bp (heatmap loses positional gradient post-peel), the L_count Gaussian area trap (L_count fights L_probe_anchor during warmstart), and the direction-handling robustness gap (V1's `abs()` convention handles direction via temporal ordering but is vulnerable to aligner errors). All three fixes adopted: Sinkhorn-OT for differentiable alignment, dynamic L_count target, bidirectional soft-min.
- **Deep Think review 4** (2026-04-20): on refined loss composition. Identified the anchoring-bias landmine in L_bp (naive `bp - bp[0]` forces velocity head to predict zero in pre-peak buffer regions, violating Stokes physics) and the per-batch-element Sinkhorn dispatch bottleneck (~1600 sequential kernel launches per step). Both fixes adopted: mean-centering for translation-invariant Wasserstein, and fully-batched Sinkhorn with `-inf` masking across padded `[B, T, max_N]` tensors. Noted `geomloss.SamplesLoss` as the production-preferred alternative to the hand-rolled implementation.
- **Deep Think review 5** (2026-04-20): on training curriculum design. Rejected the split phase-based curriculum (Option C from the brainstorm) on two grounds: (a) the peak heads are already structurally insulated from defect regions because `active_mask` is derived from GT `seg_labels`, not model predictions — the fold-contamination concern doesn't exist; (b) progressive data-filtering would cement the shared backbone representation to ignore tags on harder molecules during early phases, creating catastrophic distribution shift when those molecules are introduced. Locked in Option (D): full dataset from epoch 0, no temporal data-filtering curriculum, combined with a static quality floor at preprocessing to protect Sinkhorn from degenerate alignment cases.
- **Deep Think review 6 (full-document)** (2026-04-20): identified two critical math blockers and three runtime traps before implementation. All five adopted:
  - **L_heatmap_sharpness inversion** — the `(max_pool(m) − m)²` formulation was minimized by flat heatmaps, not peaky ones. Sign was inverted. Replaced with negative-L2-norm-under-L1-constraint (maximize L2 when L_count holds L1 fixed). See Section 6.7.
  - **L_velocity / FWHM contradiction** — the explicit tag-width velocity loss is mathematically broken in V2: during warmstart the flat boxcar heatmap gives FWHM≈1000 samples → target velocity ≈0.5 bp/sample (too low); post-peel the Dirac-limit heatmap gives FWHM≈1 sample → target velocity ≈511 bp/sample (10× physical limit, gradients explode). Dropped entirely. Velocity head remains supervised via L_bp integral constraints and wide-kernel architectural smoothness. See Section 6.4.
  - **Sinkhorn NaN trap on fully-padded rows** — `logsumexp([-inf, ...])` returns `-inf` whose gradient is `softmax([-inf, ...]) = NaN`, and `0.0 * NaN = NaN` poisons weights on the first backward pass. Fixed with a `torch.where` pattern that neutralizes fully-padded rows before `logsumexp` and restores the `-inf` mask on the output. See Section 6.2.
  - **PixelShuffle1d doesn't exist in PyTorch** — `nn.PixelShuffle` is hardcoded for 4D spatial tensors. Documented einops `rearrange` as the primary implementation, with view-based shuffle and ConvTranspose1d as fallbacks. See Section 5.5.
  - **Tensor shape / dim mismatches** — (i) channel-reduction 1x1 convs mandated after every decoder skip-concat (Section 5.5), (ii) boolean max-pool requires cast-unsqueeze-pool-squeeze-cast pattern (Section 5.8), (iii) L_count batched with `[B]` target tensor to avoid scalar broadcast failures (Section 6.3).
  - Additionally: full batching discipline documented as a requirement (Section 6.10 #8) — no `for b in range(B)` python loops anywhere in the V2 loss path.
